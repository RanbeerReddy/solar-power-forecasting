# Contributing

Thank you for considering a contribution to this project.

## Project structure

- [docs/](docs/) contains the research plan, dataset comparison, and literature review.
- [notebook/](notebook/) contains the analysis notebook.
- [report/](report/) stores the compiled report.
- [figures/](figures/) stores exported plots generated by the notebook.

## Coding style

- Prefer clear, descriptive variable names.
- Keep notebooks readable with markdown headings and short explanatory comments.
- Avoid changing the scientific methodology unless clearly documented.

## Notebook execution

1. Install dependencies from [requirements.txt](requirements.txt).
2. Place the dataset files in [data/](data/) as described in [data/README.md](data/README.md).
3. Run the notebook from top to bottom.
4. Keep the resulting figures in [figures/](figures/) for review.

## Pull request guidelines

- Open an issue first for substantial changes.
- Keep changes focused and well documented.
- Verify that the notebook still runs or explain any environment-specific limitations.
- Include a concise summary of the motivation and impact.

## Issue reporting

Please report bugs, data issues, or documentation problems with a clear description and reproduction steps.
